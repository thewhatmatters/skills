# skills

Personal [Claude Code](https://claude.com/claude-code) skills, version-controlled.

This repo lives at `~/.claude/skills/`. Each subdirectory is a skill Claude can
invoke. **Everything here is committed except secrets and generated junk** —
drop a new skill into the folder and it's tracked automatically.

### Frontend

The build → animate → audit → polish loop for web UI, plus visual sourcing.
These compose with the external `shadcn` / `next-best-practices` skills below
and carry to Tauri webviews; `polish-copy` also serves iOS.

| Skill | Status | What it does |
|-------|--------|--------------|
| [`build-ui`](build-ui/) | ✅ working | Implements UI in a project following its *existing* stack + conventions, not inventing new ones. Probes `package.json` / `tailwind.config.*` / `components.json` / `tsconfig.json` / `src/` shape, then loads only the matching references (Tailwind, shadcn/ui, vanilla-CSS, a11y). Composes with `example-skills:frontend-design` (aesthetic direction), `source-ui` (visual reference), and `automate-browser`/`example-skills:webapp-testing` (verification). Owns the *execution* axis of frontend work; aesthetic taste lives in `frontend-design`. Self-audited 0 critical / 0 important. |
| [`add-motion`](add-motion/) | ✅ working | Adds motion to web UI as a deliberate craft — entrances, staggers, gestures, scroll-linked and shared-element moves that feel premium without jank. Probes the animation stack and routes by tier: **CSS first**, the **Motion** library (ex-Framer Motion) only where it earns its bundle, **View Transitions** for route changes. Defers Motion API detail to the official `/motion` skill when installed (Motion+); enforces non-negotiable guardrails (transform/opacity only, `prefers-reduced-motion` always). No-monoculture: never adds an animation library without consent. Sibling of `build-ui`; carries to Tauri webviews unchanged. Self-audited 0 critical / 0 important. |
| [`audit-ui`](audit-ui/) | ✅ working | Layered pre-ship UI audit: **a11y** (axe-core via Playwright + a judgment checklist tools can't decide), **responsive visual review** (breakpoint screenshots Claude actually reads), **Core Web Vitals** lab budgets (Lighthouse via npx; honest about lab-vs-field — never claims INP), and **design-token drift** against the project's `DESIGN.md`. Report-only with severity-grouped findings and `file:line` evidence; fixing hands off to `build-ui`. Dimension picker with live preflight statuses; every dependency gap degrades only its own dimension. All five scripts live-verified. Self-audited 0 critical / 0 important. |
| [`polish-copy`](polish-copy/) | ✅ working | Reviews product microcopy to a luxury-grade bar in two layers: a universal craft **floor** (vague CTAs, robotic errors, dead empty states, inconsistent casing, leaked jargon) plus the project's **voice spec** — the `## Voice` section in `DESIGN.md` (house convention) or a standalone `VOICE.md`. Proposes one before→after rewrite per finding, tagged to the rule it serves; applies only what's approved, never mass-rewrites. No spec → floor-only plus an offer to bootstrap a draft `## Voice` from your existing copy (luxury-baseline seeded). Extracts from JSX/TSX/HTML **and iOS String Catalogs** (`.xcstrings`/`.strings`) — the same pass serves Swift projects. Self-audited 0 critical / 0 important. |
| [`source-ui`](source-ui/) | ✅ working | Sources real-world UI references — screens, components, user flows, and visual-style direction — by routing a design need to the right library (**Mobbin** or **Refero**) with the right tool, then returning a curated, cited set. Mobbin for fast component-level image sweeps; Refero for visual-style systems, page-level screens, and multi-step flows; built-in `WebSearch`/`WebFetch` fallback when neither MCP server is connected. Composes with `render-html` / `generate-prd`. Built by `generate-skill`; self-audited 0 critical / 0 important; live-verified. |

## External skills (composed, not vendored)

Well-maintained upstream skills are installed via [skills.sh](https://www.skills.sh),
pinned in [`skills-lock.json`](skills-lock.json), and gitignored — the same posture
as `node_modules` + a lockfile. Pinned: **`shadcn`** (shadcn/ui),
**`next-best-practices`** (vercel-labs/next-skills), **`design-md`**
(google-labs-code/stitch-skills), and **`remotion-best-practices`**
(remotion-dev/skills). Our skills defer to them through probe gates; the full
5-step convention is in [`CLAUDE.md`](CLAUDE.md).

## Setup

Some skills need API keys. Keys live in a single shared `.env` file that is
**never committed** (`.gitignore` blocks it). `.env.example` is the template:

```sh
cp ~/.claude/skills/.env.example ~/.claude/skills/.env
chmod 600 ~/.claude/skills/.env
# then fill in the keys you need
```

Resolution order (first hit wins): real env var → `~/.claude/.env` →
`~/.claude/skills/.env`. Empty values are skipped.
